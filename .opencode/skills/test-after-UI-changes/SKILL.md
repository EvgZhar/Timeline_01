# Post-UI Verification

После каждого выполненного задания (промпта), связанного с UI:

1. **Открыть приложение** через Playwright
2. **Авторизоваться:**
   - Если явно указано проверить под "гостем" — логиниться под `testuser` (`testuser@test.com` / `test1234`)
   - Если неочевидна роль или указан "админ" — логиниться под `admin` (`admin@test.com` / `admin`)
3. **Проверить**, что результат соответствует текущему заданию (ТЗ из промпта)
4. **Если результат НЕ соответствует ожиданиям:**
   - Вывести нумерованный список выявленных несоответствий
   - Уточнить у пользователя через `question` tool (с опциями "Да"/"Нет"), нужно ли всё это исправлять
   - Если пользователь отвечает "Да" — приступить к исправлению
   - Если "Нет" — зафиксировать и завершить

## Важно

- Не проверять то, что не было частью текущего задания
- При логине использовать существующие сидовые учётки
- Если приложение не открывается — сообщить об этом пользователю
