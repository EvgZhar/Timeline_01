# Post-UI Verification

После каждого выполненного задания (промпта), связанного с UI:

1. **Открыть приложение** через Playwright
2. **Авторизоваться:**
   - Если явно указано проверить под "гостем" — логиниться под `testuser` (`testuser` / `test1234`)
   - Если неочевидна роль или указан "админ" — логиниться под `admin` (`admin` / `admin`)
3. **Проверить**, что результат соответствует текущему заданию (ТЗ из промпта)
 - определить, как прийти к интересующей ситуации (надо ли сделать настройки тестовых данных), если проверка подразумевает использование объектов, обладающих особенностями, выделить такие объекты для тестирования), найдя их в БД приложения. Если  таких объектов не найдено, запросить у пользователя, предложив также автоматически создать такие объекты в рамках тестирования
 - Определить, какой элемент на форме надо активировать для проверки (если не получается определить - спросить у пользователя)
 - выполнить действия
4. **Если результат НЕ соответствует ожиданиям:**
   - Вывести нумерованный список выявленных несоответствий
   - Уточнить у пользователя через `question` tool (с опциями "Да"/"Нет"), нужно ли всё это исправлять
   - Если пользователь отвечает "Да" — приступить к исправлению
   - Если "Нет" — зафиксировать и завершить

## Важно

- Не проверять то, что не было частью текущего задания
- При логине использовать существующие сидовые учётки
- Если приложение не открывается — сообщить об этом пользователю
