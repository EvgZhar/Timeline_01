# AI-справка (YandexGPT)

## Предусловия

- API запущен (`npm run dev`)
- `.env` содержит `YANDEXGPT_API_KEY` и `YANDEXGPT_FOLDER_ID`
- Сидовые данные загружены (`npm run db:seed`)
- Пользователь `testuser` / `test1234` имеет квоту 10

## Сценарий 1: успешная генерация

1. Залогиниться как `testuser` / `test1234`
2. Открыть любое событие (клик на таймлайне или в таблице)
3. На вкладке «Основное» нажать кнопку `AI` рядом с полем «Описание»
4. Дождаться завершения спиннера
5. Проверить: в поле «Описание» появился текст в Markdown
6. Нажать «Сохранить»
7. Проверить: описание сохранилось

## Сценарий 2: лимит исчерпан

1. Установить квоту пользователю `testuser` = 1 (через админку)
2. Сгенерировать AI-справку для любого события — успех
3. Попробовать сгенерировать для другого события
4. Проверить: кнопка `AI` неактивна, тултип «Лимит запросов исчерпан»

## Сценарий 3: отображение в профиле

1. Открыть «Личный кабинет» (аватарка → профиль)
2. Проверить: отображается строка «Запросов к AI-справке: N / M»

## Сценарий 4: администрирование

1. Залогиниться как `admin` / `admin`
2. Перейти в админку (`/admin`)
3. Проверить: в таблице пользователей колонка `AI N/M`
4. Нажать «Редактировать» у `testuser`
5. Изменить поле `AI-квота` на `20`
6. Сохранить
7. Проверить: в строке пользователя отображается `AI .../20`

## Сценарий 5: сброс квот

```bash
npm run db:reset-ai-quota -w @timeline/api
```

1. Проверить лог: «Сброшены AI-квоты для N пользователей»
2. Залогиниться как `testuser`
3. Проверить в профиле: `0 / 10`
